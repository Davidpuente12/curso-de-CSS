Contexto de Apilamiento

El contexto de apilamiento consiste en la superposición de capas o elementos a lo largo del eje Z del navegador. Esto es importante para evitar que un elemento esté ocultando a otro.

Planos y ejes: el navegador está constituido de tres planos y ejes: el ancho o X; el alto o Y; y el de profundidad o Z. El eje X positivo está hacia la derecha; el eje Y positivo está hacia abajo; y el eje Z positivo está hacia el usuario. Estos son muy importantes para mover los elementos del HTML desde un punto inicial hacia un punto final.

Por defecto, todos los elementos tienen un valor auto, es decir, el orden está definido por la estructura del HTML. Los primeros elementos estarán detrás y los últimos estarán de frente. Si se establece un valor positivo, este elemento se sitúa por delante de los demás. Si se establece un valor negativo, se sitúa por detrás. Si un elemento tiene un z-index mayor a otro, estará por delante. Sin embargo, si un elemento tiene un z-index menor a otros, sus hijos nunca estarán por encima, aunque su z-index sea mayor.

z-index

controla el orden de apilamiento de los elementos en una página web, es decir, cuál se muestra por encima de otro cuando se superponen, sobre el eje z (profundidad), perpendicular a la pantalla. Funciona solo en elementos posicionados (absolute, relative, fixed, sticky) y en hijos directos de contenedores flex.

Valores Resultado
z-index: -1 El elemento estara detras de todo lo demas.
z-index: 0 Nivel base (igual que auto).
z-index: 1...+ Los elementos se posicionan de acuero al numero de mayor peso.
